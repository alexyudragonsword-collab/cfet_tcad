STACKED CMOS TCAD — User Guide

An open-source CFET / stacked-nanosheet TCAD system built on Python, DEVSIM (drift-diffusion solver), and gmsh (parametric meshing), with a desktop workbench patterned on Synopsys Sentaurus.

1. Installation

# system libraries (Debian/Ubuntu)
sudo apt-get install libopenblas0 libglu1-mesa

pip install -e '.[gui,viz]'     # GUI (PySide6) + 3D rendering (PyVista)
cfet-tcad-gui                   # launch from the project root

The package locates OpenBLAS for DEVSIM automatically; no environment variables are needed. Run pytest once (~3 min) to confirm the installation.

2. Five-minute tour

Workflow: pick a config on the left → review/edit it in Parameters → press Run → watch the row turn green in Experiments → double-click the row to open Results and Structure 3D.

2.1 The Experiments table

One row per simulation. The status chip follows the Sentaurus Workbench convention — gray queued, yellow running, green done, red failed — and the figure-of-merit columns (Vt, SS, Ion, Ioff, DIBL) are filled in automatically from the run's extraction output.

2.2 Parameters

The form is generated from the configuration schema and grouped by section. Enumerated fields (structure, mobility model, quantum model, materials) are combo boxes; every save is validated, so a typo is caught before a solver ever starts. Run always uses the current form state, so you can edit and re-run without touching files.

2.3 Results

Interactive curves re-plotted from the run's CSV (zoom/pan toolbar, log/linear toggle) plus the flattened figure-of-merit table. All four experiment types render appropriately: transfer (Id–Vg), output (Id–Vd), CFET common-gate pairs, and inverter VTC.

2.4 Structure 3D

Real VTK rendering of the device. Use the toolbar Structure button for a seconds-fast preview of the current configuration (mesh + doping, no solve — the SDE workflow), or double-click a finished experiment to load its bias-point snapshots. The combos switch the snapshot (bias point) and the colored field: Structure (materials), NetDoping (symmetric-log), Potential, Electrons/Holes (log), and Lambda_n/p (quantum potential). clip open cuts the device at the mid-plane to expose the channel.

3. Sweeps (design of experiments)

Sweep… opens the DOE dialog. Enter one parameter per line as path=v1,v2,…; multiple lines form a cartesian product, or check zip to advance the lists together as paired tuples. Points expand into individual table rows executed by a bounded process pool — rows light up green one by one:

device.l_gate_nm=12,15,18,21                    # gate-length scaling study

# zipped pair: SiGe composition sweep with the gate metal retuned
# at each composition to hold Vt constant
device.channel_material_p=Silicon,SiGe15,SiGe30,SiGe45
device.gate_workfunction_p_ev=4.72,4.655,4.59,4.525

4. Device structures

device.structureDescription
nanosheet_2d2D double-gate along-channel cross section — the standard 2D approximation of a GAA nanosheet. Fastest; ideal for methodology and sweeps.
gaa_3dFull 3D gate-all-around single nanosheet: silicon bar wrapped by an oxide shell carrying one all-around gate.
cfet_2dCFET stack: nFET sheet over pFET sheet, four gate contacts on a common gate (per-sheet metals), both devices in one coupled Newton system.
cfet_3dTwo stacked 3D GAA nanosheets with a common gate — the full CFET.

Key geometry knobs (all nm): l_gate_nm, t_si_nm, t_ox_nm, l_sd_nm, sheet_width_nm, t_gap_nm (CFET inter-sheet gate metal). Doping: sd_doping_cm3, channel_doping_cm3, junction_lambda_nm (gaussian junction tail). Vt is set by the gate metal: gate_workfunction_ev (single device) or gate_workfunction_n_ev/_p_ev (CFET).

5. Physics models

SettingOptionsNotes
physics.mobility_model const, doping, doping_vsat, lombardi_vsat doping_vsat: Caughey–Thomas doping dependence + velocity saturation. lombardi_vsat adds CVT vertical-field surface scattering (element-based assembly, 2D and 3D) — expect roughly −25% linear Ion on the default device.
physics.quantum_model none, density_gradient Bohm quantum potential for the transport carrier with an oxide-barrier Robin interface condition; produces volume inversion and a positive Vt shift (~+15–20 mV at tsi=5 nm). Strength calibrates via dg_gamma_n/p.
combinedlombardi_vsat + density_gradient The "full physics" pairing Sentaurus uses by default on advanced-node devices; effects compose near-multiplicatively.
device.channel_material
(_n/_p for CFET)		 Silicon, SiGeNN SiGeNN = strained Si1-xGex with NN = Ge percent (0–50), parameters interpolated continuously. Retune the pFET gate metal when changing composition (the bandgap moves the band reference).

7. Command line

cfet-tcad run <config.yaml> [-o outdir]      # full pipeline
cfet-tcad sweep <config> -p PATH=V1,V2 [-p ...] [--zip] [-j N]
cfet-tcad structure <config> [-o dir] [--png img.png]   # no-solve preview
cfet-tcad mesh <config>                      # gmsh mesh only

Each run writes to its output directory: idvg.csv / idvd.csv / vtc.csv (all bias points), *.png (curves), fom.json (extracted figures of merit), and vtk/ (field snapshots — also openable in VisIt/ParaView via the .pvd/.visit index, with the sweep bias as the timestep).

8. Tips & troubleshooting

  • Typical runtimes — 2D Id–Vg: ~1–2 min; 3D: a few minutes; lombardi_vsat or density_gradient add ~30–50%. Structure previews take seconds.
  • Convergence — bias ramps halve their step automatically; the quantum and CVT models ramp in via internal homotopy. If a point still fails, coarsen the sweep step or the mesh first.
  • Every experiment is a separate OS process (DEVSIM keeps global state), so a crash never takes the workbench down; check the row's run.log-equivalent in the log console.
  • Vt tuning — use the gate workfunction, not channel doping: these channels are essentially undoped, as in real GAA/CFET devices.
  • Short-channel limits — with tsi=5 nm the electrostatic scaling limit is around Lg ≈ 12–15 nm; below that the extractor honestly reports NaN because the device no longer turns off past the Icrit criterion.
  • Headless / CI — the GUI runs under QT_QPA_PLATFORM=offscreen; 3D renders fall back to OSMesa or xvfb-run.
