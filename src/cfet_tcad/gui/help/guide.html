STACKED CMOS TCAD — User Guide

An open-source CFET / stacked-nanosheet TCAD system built on Python, DEVSIM (drift-diffusion solver), and gmsh (parametric meshing), with a desktop workbench patterned on Synopsys Sentaurus.

1. Installation

# system libraries (Debian/Ubuntu)
sudo apt-get install libopenblas0 libglu1-mesa

pip install -e '.[gui,viz]'     # GUI (PySide6) + 3D rendering (PyVista)
cfet-tcad-gui                   # launch from the project root

The package locates OpenBLAS for DEVSIM automatically; no environment variables are needed. Run pytest once (~3 min) to confirm the installation.

2. Five-minute tour

Workflow: the current config folder path sits above the YAML list on the left (change it with Open in the menu bar). Right-click a YAML → Add to register it as a pending experiment row → press that row's Run button (or Run All in the toolbar) → watch the row turn green → double-click the row to load Results (bottom-left) and Structure 3D (bottom-right). All three panes live in one window, separated by draggable splitters.

2.1 The Experiments table

One row per simulation, each row driving itself through the buttons in its Actions cell: Run (start, or re-run a finished row in place — same output folder, results overwritten), Stop, Sweep (expand a DOE grid over this row's config), and Structure (fast mesh preview, no solve). The status chip follows the Sentaurus Workbench convention — light-gray pending (added, waiting for Run), gray queued, yellow running, green done, red failed, gray-blue stopped — and the figure-of-merit columns (Vt, SS, Ion, Ioff, DIBL) are filled in automatically from the run's extraction output. The toolbar's Run All starts every pending/stopped/failed row; Stop All halts everything.

2.2 The parameter editor

Double-click a YAML in the left list (or right-click → Edit) to pop up its parameter editor. The form is generated from the configuration schema and grouped by section. Enumerated fields (structure, mobility model, quantum model, materials) are combo boxes; every save is validated, so a typo is caught before a solver ever starts. Save overwrites the opened file; Save As… writes the edits to a new YAML. The right-click menu also offers Copy… (duplicate under a new name) and Delete (confirmed file removal).

2.3 Results

Interactive curves re-plotted from the run's CSV (zoom/pan toolbar, log/linear toggle) plus the flattened figure-of-merit table. All experiment types render appropriately: transfer (Id–Vg), output (Id–Vd), CFET common-gate transfer and output (Id–Vd) pairs, and inverter VTC.

2.4 Structure 3D

Real VTK rendering of the device. Use a row's Structure button for a seconds-fast preview of that experiment's configuration (mesh + doping, no solve — the SDE workflow), or double-click a finished experiment to load its bias-point snapshots. The combos switch the snapshot (bias point) and the colored field: Structure (materials), NetDoping (symmetric-log), Potential, Electrons/Holes (log), and Lambda_n/p (quantum potential). clip open cuts the device at the mid-plane to expose the channel.

3. Sweeps (design of experiments)

A row's Sweep button opens the DOE dialog over that row's config. Enter one parameter per line as path=v1,v2,…; multiple lines form a cartesian product, or check zip to advance the lists together as paired tuples. Points expand into individual pending table rows — press Run All (or per-row Run) to execute them through the bounded process pool, and the rows light up green one by one:

device.l_gate_nm=12,15,18,21                    # gate-length scaling study

# zipped pair: SiGe composition sweep with the gate metal retuned
# at each composition to hold Vt constant
device.channel_material_p=Silicon,SiGe15,SiGe30,SiGe45
device.gate_workfunction_p_ev=4.72,4.655,4.59,4.525

Import CSV… fills the dialog from a design-point table (rows = runs, dotted config paths as headers) — an edited sweep_summary.csv from a previous sweep works as-is, its status/FOM columns are ignored. The CLI equivalent is cfet-tcad sweep base.yaml --points doe.csv.

4. Device structures

device.structureDescription
nanosheet_2d2D double-gate along-channel cross section — the standard 2D approximation of a GAA nanosheet. Fastest; ideal for methodology and sweeps.
gaa_3dFull 3D gate-all-around single nanosheet: silicon bar wrapped by an oxide shell carrying one all-around gate.
cfet_2dCFET stack: nFET sheet over pFET sheet, four gate contacts on a common gate (per-sheet metals), both devices in one coupled Newton system.
cfet_3dTwo stacked 3D GAA nanosheets with a common gate — the full CFET.

Key geometry knobs (all nm): l_gate_nm, t_si_nm, t_ox_nm, l_sd_nm, sheet_width_nm, t_gap_nm (CFET inter-sheet gate metal). Doping: sd_doping_cm3, channel_doping_cm3, junction_lambda_nm (gaussian junction tail). Vt is set by the gate metal: gate_workfunction_ev (single device) or gate_workfunction_n_ev/_p_ev (CFET).

Design import — external meshes. Set structure: external to simulate a user-supplied gmsh MSH 2.2 ASCII mesh (units: cm). The device.external section maps the file's physical groups onto regions / contacts / interfaces and picks a doping recipe per silicon region (lateral_sd analytic profile, uniform, or a raw DEVSIM expression in x/y/z):

device:
  structure: external
  external:
    mesh_file: my_device.msh      # relative to the config file
    dimension: 2
    regions:    {bulk: Silicon, gox: Oxide}
    contacts:   {source: bulk, drain: bulk, gate: gox}
    interfaces: {si_ox: [bulk, gox]}
    doping:
      bulk: {profile: uniform, donors_cm3: 1.0e17, acceptors_cm3: 0}

STEP (CAD) import. A .step/.stp assembly can be converted into a simulation-ready mesh: STEP files carry geometry only, so a small mapping spec assigns every solid to a region (selector: label regex on the CAD part name, volume tag, or bbox), places the contacts (bbox), and sets unit_cm (1 CAD unit in cm — nm: 1e-7, µm: 1e-4, mm: 0.1). In the GUI, .step files appear in the config browser; double-click (or right-click → Convert to mesh…) opens a dialog prefilled with the file's volume table and a spec template — Convert writes the .msh plus a runnable starter config. On the command line: cfet-tcad import-step model.step --list prints the volume table, cfet-tcad import-step spec.yaml converts. Solids are boolean-fragmented so touching regions mesh conformally, and meshing happens in CAD coordinates (the finished mesh is scaled to cm), keeping nm-scale features clear of the CAD kernel's tolerance. Two complete worked examples ship in configs/: paper_fbc_cfet_demo.step (the paper's dual-fin FBC CFET) and paper_sbc_cfet_demo.step (the dual-sheet SBC CFET), each with its mapping spec and a runnable *_run.yaml — convert the STEP first, then Add/Run the config.

Design export. The Structure 3D tab's Export… button (or cfet-tcad structure cfg.yaml --stl dev.stl --obj dev.obj) writes the device geometry as STL/PLY/VTP surface meshes or a material-colored OBJ+MTL for CAD and presentation tools. Simulation results always land as CSV, fom.json, VTK (VisIt/ParaView), and the gmsh .msh alongside.

Multi-channel replication (cfet_3d): n_fins / fin_pitch_nm mesh several fins side by side per device (paper-style fin arrays); n_stacked_sheets / sheet_pitch_nm stack several nanosheets vertically. Replicas share the device's regions and contacts, so everything downstream works unchanged. n_sheets stays a pure current multiplier (no geometry).

ParaView: open vtk/*.pvd directly (timesteps = bias values animate the sweep); the repo ships examples/paraview_macro_fig8.py, a one-click macro for a clipped, mobility-colored view with optional OSPRay ray tracing.

5. Physics models

SettingOptionsNotes
physics.mobility_model const, doping, doping_vsat, lombardi_vsat doping_vsat: Caughey–Thomas doping dependence + velocity saturation. lombardi_vsat adds CVT vertical-field surface scattering (element-based assembly, 2D and 3D) — expect roughly −25% linear Ion on the default device.
physics.quantum_model none, density_gradient Bohm quantum potential for the transport carrier with an oxide-barrier Robin interface condition; produces volume inversion and a positive Vt shift (~+15–20 mV at tsi=5 nm). Strength calibrates via dg_gamma_n/p.
combinedlombardi_vsat + density_gradient The "full physics" pairing Sentaurus uses by default on advanced-node devices; effects compose near-multiplicatively.
device.channel_material
(_n/_p for CFET)		 Silicon, SiGeNN SiGeNN = strained Si1-xGex with NN = Ge percent (0–50), parameters interpolated continuously. Retune the pFET gate metal when changing composition (the bandgap moves the band reference).

6. Experiments

simulation.typeWhat it doesKey settings
idvgGate sweep at each drain bias in vd; extracts Vt (constant-current and max-gm), SS, Ion/Ioff, and DIBL when two Vd values are given.vd, vg_start/stop/step
idvdDrain sweep at each gate bias in vg (output characteristics).vg, vd_start/stop/step
cfet_idvgCFET common-gate sweep under CMOS-pair bias; one coupled solve yields both transfer curves and both FOM sets.vdd, vg_*
cfet_idvdCFET output characteristics: at each fixed common-gate bias in vg, sweep the drains (nFET drain 0→Vd, pFET drain Vdd→Vdd−Vd) for both devices' Id–Vds.vdd, vg, vd_*
cfet_vtcInverter voltage transfer characteristic: Vout is a floating circuit node solved self-consistently (mixed device/circuit). Extracts VM, gain, noise margins.vdd, vg_*

While a row is running its Status cell shows a live progress percentage (measured bias points / total). Besides the per-row buttons, right-clicking a row offers Stop (kills that run; the row turns gray-blue "stopped"), Remove from list (drops the row, files on disk are kept), and Open results folder. The toolbar Stop All still stops everything at once.

7. Command line

cfet-tcad run <config.yaml> [-o outdir]      # full pipeline
cfet-tcad sweep <config> -p PATH=V1,V2 [-p ...] [--zip] [-j N]
cfet-tcad structure <config> [-o dir] [--png img.png]   # no-solve preview
cfet-tcad mesh <config>                      # gmsh mesh only

Each run writes to its output directory: idvg.csv / idvd.csv / vtc.csv (all bias points), *.png (curves), fom.json (extracted figures of merit), and vtk/ (field snapshots — also openable in VisIt/ParaView via the .pvd/.visit index, with the sweep bias as the timestep).

8. Tips & troubleshooting

  • Typical runtimes — 2D Id–Vg: ~1–2 min; 3D: a few minutes; lombardi_vsat or density_gradient add ~30–50%. Structure previews take seconds.
  • Convergence — bias ramps halve their step automatically; the quantum and CVT models ramp in via internal homotopy. If a point still fails, coarsen the sweep step or the mesh first.
  • Every experiment is a separate OS process (DEVSIM keeps global state), so a crash never takes the workbench down; check the row's run.log-equivalent in the log console.
  • Vt tuning — use the gate workfunction, not channel doping: these channels are essentially undoped, as in real GAA/CFET devices.
  • Short-channel limits — with tsi=5 nm the electrostatic scaling limit is around Lg ≈ 12–15 nm; below that the extractor honestly reports NaN because the device no longer turns off past the Icrit criterion.
  • Headless / CI — the GUI runs under QT_QPA_PLATFORM=offscreen; 3D renders fall back to OSMesa or xvfb-run.

All numbers shown in this guide are actual outputs of the bundled example configurations, reproduced by the test suite.
