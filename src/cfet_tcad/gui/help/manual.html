STACKED CMOS TCAD — Software Manual

Version 0.5 · Author: Yu Rui · Open source (Apache-2.0)

An open-source, device-level TCAD system for CFET and stacked gate-all-around (GAA) nanosheet transistors, built on Python, DEVSIM (drift-diffusion solver) and gmsh (meshing), with a desktop workbench patterned on the Synopsys Sentaurus tool chain. This manual summarizes what the software does, where it applies, how it compares to mainstream tools, and where its limits are. For step-by-step operating instructions see Help → User Guide.

1. Overview

STACKED CMOS TCAD solves the semiconductor drift-diffusion equations on 2D/3D meshes to predict the electrical characteristics of advanced CMOS devices — nanosheet GAA FETs, complementary FETs (CFET), and vertically stacked devices — from a small YAML description or an imported CAD/mesh file. It targets the 3 nm node and beyond and is intended for research, methodology development, and teaching.

2. Features

Device structures

  • nanosheet_2d — 2D double-gate along-channel cross section (the standard fast 2D GAA approximation)
  • gaa_3d — full 3D single-nanosheet gate-all-around
  • cfet_2d / cfet_3d — complementary FET stacks (nFET over/under pFET)
  • Multi-channel replication — several fins side by side (n_fins) or several sheets stacked (n_stacked_sheets) per device
  • external — import a user-supplied gmsh MSH 2.2 mesh, or a STEP (.step) CAD assembly converted in-app

Physics

  • Drift-diffusion with Scharfetter–Gummel discretization
  • Mobility: Caughey–Thomas (doping), CVT/Lombardi vertical-field degradation, high-field velocity saturation
  • Density-gradient quantum correction (confinement Vt shift, element-level current)
  • SiGe(x) composition interpolation; per-surface-orientation mobility calibration (mobility_scale_n/p)

Experiments & extraction

  • Transfer (Id–Vg), output (Id–Vd), CFET common-gate transfer, and inverter voltage-transfer characteristic (mixed device/circuit Newton)
  • Figures of merit: Vt (constant-current and max-gm), subthreshold swing, DIBL, Ion/Ioff, inverter gain and noise margins

Workflow & platform

  • Sentaurus-Workbench-style Experiments table with per-row Run/Stop/Sweep/Structure, live progress %, parallel sweeps / DOE, and CSV design-point import
  • Interactive 3D VTK rendering; geometry import/export (STEP in; STL/PLY/VTP/OBJ out)
  • Cross-platform: Linux and a standalone Windows executable (MKL PARDISO on Windows, OpenBLAS+UMFPACK on Linux)
  • Open source under Apache-2.0; scriptable Python API and CLI

3. Scope of application

Intended for:

  • Device-level TCAD of advanced CMOS (GAA nanosheet, CFET, stacked devices) at the 3 nm node and beyond
  • Methodology development, design-of-experiments sweeps, and architecture trade-off studies (e.g. fin- vs sheet-based CFET)
  • Teaching drift-diffusion transport and quantum correction with a transparent, open codebase

Not intended for:

  • Production process- or layout-level sign-off
  • Interoperating with proprietary vendor formats (TDR/BND/CMD, GDSII/OASIS)
  • Stress-engineering or ring-oscillator circuit sign-off

4.2 Device-level reproduction result

The fin-based (FBC) vs sheet-based (SBC) CFET comparison was reproduced against Jiang et al. (Applied Materials), "Complementary FET Device and Circuit Level Evaluation Using Fin-Based and Sheet-Based Configurations Targeting 3nm Node and Beyond", whose data come from a commercial (Sentaurus-class) TCAD flow. Using a constant-current Ion at Ioff=1 nA (the paper's convention), all four current deltas match the paper in direction:

ConfigurationΔIon,npaper ΔIon,ppaper
FBC (2 fins, 5×18 nm)(ref)(ref) (ref)(ref)
SBC (2 sheets, 18×5 nm)+7.7%+10% −9.1%−5%
SBC wide (2 sheets, 31×5 nm)+48.9%+73% +28.0%+47%

Subthreshold swing lands at 72–76 mV/dec across the three configurations. Absolute currents are not directly comparable (the orientation mobility ratios use literature values rather than the paper's per-curve sub-band BTE calibration), but the device-level trends reproduce correctly: SBC nMOS leads FBC thanks to the (100) electron-mobility advantage, SBC pMOS trails on hole mobility, and a wider sheet lifts both.

4.3 Numerical cross-validation

Beyond the paper comparison, the solver is checked internally to bit-level agreement: disconnected device regions solve block-diagonally (proven exact), and replicated channels carry exactly N× the single-channel current (relative error <10-6). On Windows the same MKL PARDISO direct solver used by commercial tools is employed.

5. Limitations

  • No layout import (GDSII/OASIS): this is a device-level tool, upstream of process simulation.
  • No proprietary-format interop: Sentaurus TDR/BND/CMD files are not read or written.
  • No stress mobility model: strain effects are approximated by manual mobility calibration (mobility_scale_n/p) rather than a physical stress solver.
  • No parasitic RC extraction or ring-oscillator transient: the circuit-level capability stops at the inverter VTC (mixed-mode).
  • 3D fins are approximated as a rotated GAA (four-sided wrap-around) rather than a true tri-gate on bulk.
  • Quantum correction is available but off by default in 3D for cost; enable it explicitly when confinement Vt shift matters.
  • Mobility orientation ratios use literature values, not a per-curve BTE calibration — relative trends are reliable, absolute currents are indicative.
